# Contributing to AquaEcho

Thank you for your interest in contributing to AquaEcho! Your contributions help improve this project and make it better for everyone.

## How to Contribute

### 1. Reporting Issues
- If you find a bug or have a feature request, please check if an issue already exists in the [issue tracker](https://github.com/AdityaSeth777/AquaEcho/issues).
- If not, open a new issue with a clear and descriptive title.
- Provide steps to reproduce the bug (if applicable) and relevant screenshots or logs.

### 2. Submitting Pull Requests
- Fork the repository and create a new branch (`feature/your-feature` or `fix/your-bug`).
- Ensure your code follows the project's style guidelines and best practices.
- Write clear commit messages and reference related issues where applicable.
- Test your changes before submitting.
- Submit a pull request and provide a detailed description of the changes.

### 3. Code Guidelines
- Follow the existing coding style of the project.
- Keep changes small and focused.
- Document new functions, components, and major changes.

### 4. Commit Message Guidelines
A well-structured commit message makes it easier to understand the purpose of the changes. Follow this format:

**Format:**
```
<type>(scope): <short description>

[Optional] Longer description explaining what was changed and why.

[Optional] References to issues or PRs (e.g., Closes #123, Fixes #456)
```

**Example:**
```
feat(ui): add parallax scrolling to homepage

Implemented smooth parallax scrolling using react-native-reanimated to enhance user experience.

Closes #25
```

**Common Commit Types:**
- **feat**: A new feature
- **fix**: A bug fix
- **docs**: Documentation updates
- **style**: Code style changes (formatting, missing semicolons, etc.)
- **refactor**: Code restructuring without functionality changes
- **perf**: Performance improvements
- **test**: Adding or updating tests
- **chore**: Maintenance tasks, such as updating dependencies

### 5. Reviewing and Feedback
- Be open to feedback and ready to make improvements.
- Review other contributors' pull requests and provide constructive feedback.

### 6. Community and Support
- Join discussions in the project repository and engage with other contributors.
- Respect the [Code of Conduct](CODE_OF_CONDUCT.md).

We appreciate your contributions and look forward to building AquaEcho together!
